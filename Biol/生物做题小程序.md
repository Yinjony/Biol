# 需求文档

适用平台：web网址/app.apk

版本：V0.1

## 技术栈

前端：Uni-app+Vue3+uView/uni-ui

状态管理：Pinia

后端：Node.js+TS

数据库：PostgreSQL

OCR：使用一个OCR接口或者使用deepseek-OCR

## 核心功能

### 题库管理

对题的增删改查操作，题只有两种选择和判断

1、上传题目

使用OCR或者读取word文档，然后识别到内容后，对内容进行判断，是判断题还是选择题，然后自动进行填入；

也可以用户自己去填写题目和选项；

然后答案需要用户自己去填

2、修改题目

选择题，用户可以改题目，改选项，改答案；

判断题，用户可以改题目，改答案；

3、删除题目

删除就好，但是需要添加一个弹窗，确保用户是真的想删除题目

4、查询题目

为了方便修改题目而进行查询，对题目要求内容进行查询，使用模糊匹配的方式

### 开始做题

从数据库里随机筛选x道题（10/25/50）

然后开始做题，题目是往下滑的全部浏览，做一道题，立马判断对错，不能修改答案

全部做完后，点击提交按钮，就可以结束做题，但是要判断是否全部做完，如果有没做的题，就弹窗显示没做题的题号，做完了就显示做对题数/做错题数

## 数据库

````sql
-- 1. 启用模糊匹配扩展
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- 2. 创建枚举类型
CREATE TYPE question_type AS ENUM ('CHOICE', 'JUDGE');

-- 3. 建表
CREATE TABLE "Question" (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    type question_type NOT NULL,
    content TEXT NOT NULL,
    options JSONB,  -- 选择题必填，判断题为 null
    answer VARCHAR(10) NOT NULL,
    "createdAt" TIMESTAMPTZ DEFAULT NOW(),
    "updatedAt" TIMESTAMPTZ DEFAULT NOW()
);

-- 4. 核心索引：支持高性能中文模糊查询
CREATE INDEX idx_question_content_trgm ON "Question" USING gin (content gin_trgm_ops);
```
````
